There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

## Explanation :
### Step 1 : 
first of all, I scanned the localhost for open ports from 31000 to 32000 with nmap with the command :
`nmap -p31000-32000 localhost` .
I got few ports, 
### Step 2 : 
Then, I used `openssl s_client`  to test SSL on each open port using the command: 
```bash
openssl s_client -connect localhost:<port>
```
If it didn't returned any certificate, then it's not using SSL protocol, but the second last port returned me some kind of certificate.

### Step 3 : 
Then, i sent the password of the current level to that port using the command : 
```bash
openssl s_client -connect localhost:31790 -quiet < /etc/bandit_pass/bandit16
```
here, 
-  `31790` is the port which uses SSL
- `< /etc/bandit_pass/bandit16` pipes the password of current level to feed that port.

### Finally : 
I got the RSA key, which i mentioned above.
